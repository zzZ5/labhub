# LabHub 前端 UI 优化 TODO

> 审计日期：2026-07-17  
> 审计范围：`frontend/src` 全部路由、布局、公共组件、公开门户和内部平台  
> 实测视口：桌面端 1440 × 1000，手机端 390 × 844  
> 数据环境：公开端使用当前开发数据，内部端使用项目 Playwright 模拟数据

## 1. 优化目标

- [x] 统一为简约、克制、学术、生态的中国农业大学实验室视觉风格。
- [x] 公开门户突出团队研究、成果和成员，不呈现互联网产品或数据大屏气质。
- [x] 内部平台提高信息密度和操作效率，减少重复标题、重复卡片和无效留白。
- [ ] 桌面、平板和手机采用适合各自场景的布局，不再仅依靠单列堆叠完成移动端适配。
- [ ] 保持现有 API、权限、上传、预览、分页和业务逻辑不变。

## 2. 审计结论

### P0：优先修复的可用性问题

- [x] 修复成员管理桌面端操作区横向拥挤，当前末尾操作会贴边或被裁切。
- [x] 重做学生档案手机端主从布局，不能先展示完整学生列表，再把当前档案堆到列表下方。
- [x] 重做门户内容手机端栏目导航和编辑区，避免统计标签、栏目按钮、长表单连续占满多个屏幕。
- [x] 统一内部平台页面标题：顶部栏已有页面名时，内容区不再重复同名大横幅。
- [x] 内部资料手机端改为紧凑资料行或信息列表，避免每份资料占一个高卡片并重复显示所有元数据。
- [ ] 为所有长标题、长文件名、长邮箱和多操作按钮定义截断、换行和操作菜单规则。
- [ ] 核对 320px、360px、390px 宽度下的导航、表单、分页、标签栏和弹窗，不允许依靠 `overflow-x: hidden` 掩盖内容裁切。

### P1：影响全站一致性的主要问题

- [x] 建立唯一的公开端页面标题体系，统一列表页、详情页、首页模块的字号、引导线、间距和背景。
- [x] 建立唯一的内部端页面标题体系，统一“标题 + 摘要 + 主操作 + 辅助操作”的结构。
- [x] 减少首页和内部平台的卡片数量，页面分区优先使用留白、细分割线和浅背景带。
- [x] 减少重复浅绿渐变。生命绿只用于当前状态、主按钮、链接和重点数字。
- [ ] 统一卡片圆角、边框、阴影、内边距和 hover，不再由每个页面单独定义近似样式。
- [ ] 统一按钮层级，避免实心绿、描边绿、文字链接和 Element Plus 默认按钮混用时权重不清。
- [ ] 统一搜索、筛选、分页和批量操作工具栏，桌面与手机使用不同排布规则。
- [ ] 统一图片比例、空图片占位、焦点位置和加载状态，避免大面积“暂无图片”色块抢占页面。
